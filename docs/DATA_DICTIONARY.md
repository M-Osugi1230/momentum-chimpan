# Momentum Chimpan Data Dictionary

Last updated: 2026-07-12

## Contract rules

Every dataset belongs to one persistence class:

1. **Production state** — updated by the complete daily workflow and recoverable from sealed snapshots.
2. **Governance source** — human-reviewed configuration, registration, catalog, or approval data.
3. **Signed compact status** — derived evidence summary with integrity hashes and narrow publisher authority.
4. **Regenerable output** — workbook, log, or diagnostic that can be recreated from coherent inputs.
5. **Artifact-only research evidence** — large or raw research data retained outside the repository.
6. **External cache** — refreshable source data that is nevertheless included in operational recovery.

A new persisted file must define its class, producer, natural key, validation, retention, recovery behavior, and consumers before activation.

## Production state

### `data/momentum_daily_ranking.csv`

| Property | Contract |
|---|---|
| Purpose | Daily full scanned-universe ranking and change history |
| Producer | Core daily analysis in `main.py` |
| Update | Successful full production run |
| Natural key | `date + code` |
| Duplicate rule | Exactly one canonical row per date and code |
| Persistence | Git plus sealed state snapshot |
| Recovery | Restore latest fingerprint-compatible sealed snapshot |
| Consumers | Daily change detection, lifecycle, research workflows, Forward Evidence |

Core fields include:

- `date`, `rank`, `code`, `name`, `score`;
- `return_20d`, `volume_ratio`;
- `ytd_high_flag`, `ytd_high_streak`, `ytd_high_count`;
- `is_top100`, `is_new_entry`, `rank_change`, `is_rising_fast`;
- `is_best_rank`, `top30_streak`;
- additional current lifecycle, sector, priority, provenance, or strategy-stamp fields generated by the application.

Freshness rule: a persisted production row must correspond to the intended market date and current governed strategy fingerprint. A limited-symbol verification run must not replace production history.

## Governance sources

### `config.yaml`

Production configuration for included markets, ETF/REIT exclusions, liquidity and price thresholds, ranking output counts, lookback, and data paths.

Mutation requires a reviewed production PR. A change may affect the strategy fingerprint.

### `research/evidence_catalog.yaml`

Canonical machine-readable record of evidence precedence, study chronology, current research status, current production weight, and current decision.

Current governing decision is `HOLD_UNCHANGED_PENDING_FORWARD_EVIDENCE`.

### `research/volume_component_forward_evidence.yaml`

Pre-registration for the live prospective volume-component study, including cutoff date, execution model, horizons, costs, sample gates, and governance flags.

Acceptance gates must not be changed after results are observed without a separately registered study.

### `research/strategy_approvals.yaml`

Human-authored approval or rejection records bound to exact strategy fingerprints, evidence hashes, review-packet hashes, reviewer identity, timestamp, and scope.

Automatic activation is prohibited.

## Signed compact status

### `data/volume_component_forward_status.json`

| Property | Contract |
|---|---|
| Purpose | Small repository-visible summary of prospective evidence progress |
| Producer | Isolated `Publish Volume Component Forward Status` workflow |
| Source | Exact successful Forward Evidence artifact from `main` |
| Update rule | Commit only when semantic evidence changes |
| Allowed writer | Isolated publisher workflow |
| Integrity | `evidence_fingerprint` and full `status_sha256` |
| Consumers | Daily Summary/email/Research Evidence sheet and manual review packet |

Required content includes:

- study and status versions;
- eligible signal cutoff;
- strategy fingerprint;
- evidence origin and entry model;
- evidence status and sample adequacy;
- 10- and 20-session counts, paired dates, differences, p-values, and confidence intervals;
- source run and input hashes;
- no-automatic-change and research-only flags.

Invalid, missing, or tampered status must result in a safe warning, zero-progress display, and unchanged 15-point weight.

## Regenerable outputs

### `output/daily_report.xlsx`

Human-facing daily workbook. It is an artifact, not the canonical history database.

Current core sheets include ranking, changes, market state, universe, errors, operational/research transparency, and any current application-defined paper or analysis sheets.

### `output/run.log`

Complete daily execution log retained in the run artifact.

### Operational diagnostics

- `output/state_maintenance.json`
- `output/evidence_stamp_audit.json`
- `output/recovery_snapshot_audit.json`
- `output/ops_notification.txt`

These diagnose the run and can be regenerated or retained as artifacts. They are not standalone production history.

## Artifact-only research evidence

Examples include:

- historical price panels;
- replay signals and ledgers;
- forward signals and outcomes;
- bootstrap samples and distributions;
- detailed research workbooks;
- complete manifests and provenance bundles;
- manual review workbooks.

A compact, reviewed result may be committed under `research/results/`, but raw or large evidence should remain in Actions artifacts.

## Data-quality rules to implement

Issue #71 will formalize per-stock grades:

| Grade | Meaning | Priority effect |
|---|---|---|
| A | Complete and current | Eligible for all priority classes |
| B | Minor non-blocking warning | Eligible with warning |
| C | Material caution | Cannot become A priority |
| D | Unreliable/reference only | Exclude from priority candidates |

Checks include latest trading date, price/volume completeness, missing sessions, stale price, abnormal movement, possible corporate action, identity consistency, liquidity, and retrieval source/result.

## Schema-change policy

A schema change requires:

1. backward-compatibility or migration plan;
2. natural-key and duplicate-rule review;
3. snapshot/recovery compatibility review;
4. consumer update list;
5. test coverage;
6. documentation update;
7. explicit note if the strategy fingerprint changes.
